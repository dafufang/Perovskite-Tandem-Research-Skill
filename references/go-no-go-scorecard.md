# Go/No-Go Scorecard

Use for R&D decisions. Adjust weights if the user provides different priorities.

| Criterion | Weight | Score 1 | Score 3 | Score 5 |
|---|---:|---|---|---|
| Efficiency / energy yield | 15% | marginal improvement | credible improvement | strong certified/yield advantage |
| Stability path | 20% | weak short tests | medium evidence | strong MPP/module-relevant evidence |
| Process fit | 15% | incompatible | possible with changes | clear fit with available process |
| Scale-up path | 15% | only tiny cells | minimodule evidence | credible module/manufacturing path |
| Cost potential | 10% | high cost/low yield | uncertain | plausible cost advantage |
| Competitive/IP risk | 10% | crowded and blocked | manageable | differentiated or open path |
| Certification path | 5% | unclear | possible | clear independent validation path |
| Application fit | 10% | weak fit | moderate fit | strong fit to target application |

## Interpretation
- 4.0-5.0: Go, if key risks are actively managed.
- 3.0-3.9: Conditional Go, define validation gates.
- 2.0-2.9: Watch, monitor and run small experiments only.
- <2.0: No-Go unless strategic reasons override.

Always include the top 3 assumptions that could change the decision.
