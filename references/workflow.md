# Workflow

## Select the task mode

Use the user's wording, attachments, and requested output to select one or more modes.

| User intent | Mode | Primary references |
|---|---|---|
| Uploaded paper, named paper, DOI, title, or "this paper" | paper-review | paper-review-template.md, metrics-glossary.md, stability-assessment.md, key-papers.md |
| Latest progress, trends, hotspots, state of field | research-scan | literature-map.md, literature-update-rules.md, source-priority.md |
| Compare routes or process options | route-evaluation | technology-routes.md, route-evaluation-framework.md, go-no-go-scorecard.md |
| Internal project choice, roadmap, Go/No-Go, investment | rd-decision | rd-decision-framework.md, go-no-go-scorecard.md, experiment-planning.md |
| Design experiments or reproduce a paper | experiment-planning | experiment-planning.md, stability-assessment.md, metrics-glossary.md |
| Company/institution, records, modules, production, commercialization | competitor-watch | competitor-watchlist.md, source-priority.md, literature-update-rules.md |

## Mode priority

If multiple modes apply, use this order:
1. Verify current facts if the question is time-sensitive.
2. Classify the research branch and technology route.
3. Analyze technical details and evidence strength.
4. Translate findings into R&D implications.
5. Propose experiments or decision criteria.

## Ask or assume

If equipment, target application, budget, or timeline are missing, do not block unless the decision depends critically on them. State assumptions and provide a default recommendation. For example: "Assuming the target is high-efficiency crystalline-silicon-compatible modules...".

## Output language

Default to Chinese. Use English when the user asks in English or requests English.
